Static Web Application
Displays very little content as is not very flexible. Usually developed in HTML and CSS. To make changes to it, you will have to rework the HTML code.
Dynamic Web Application
Much more technically complex. To make changes in it administrators will have to work in CMS to update. Contents can be updated without accessing the server.

The process of website development

Information Gathering

Gather information that answers your question and find the target audience the website caters to with a questionnaire or survey.

Planning

In this stage, an outsource web development company creates a sitemap or mock-up that explains the relationship between different areas of your website.

Design

All the visual elements of the website like images and videos are added. The layout will give an idea as to how the visual structure will appear.

Content Writing

If you wish to convey certain ideas to your target audience, use catchy lines to enthrall your target audience.

Coding

Usually, the home page is created first and then other pages are added later according to the hierarchy set up by the sitemap that was formulated.

Testing and Launching

This is the place where you make sure that none of your links are broken. Using a code validator also makes sure that your code follows a web standard.

Maintenance and Update

Website development should be treated as a service. This is why it is imperative to keep the clients of an outsource web development team always satisfied.
